# 🔥 Flashphoto (阅后即焚) 

基于 Cloudflare 生态（Worker + R2 + KV）打造的**加密“阅后即焚”图片分享应用**。  
无需任何构建工具，无需服务器，单文件直接部署，为用户提供极致的隐私保护。

## ✨ 核心特性

- **🛡️ 加密 (AES-GCM)**：图片在后端内存中被动态生成的专属密钥进行流式加密后才会存入 R2 存储桶。即使是系统管理员进入控制台后台，下载的也只是一堆无法打开的乱码。
- **👁️ 阅后即焚防截屏**：长按屏幕查看，松手、切换后台或失去焦点 **（防 PC 端截图软件效果不佳，如微信可以截图）** 立即触发底层焚毁。全局禁用右键，拦截 `PrintScreen` 及 `Ctrl+S` 等快捷键。
- **📱 移动端适配 & 暗黑模式**：内置原生级触控体验，自动响应系统深色/浅色模式，并支持手动无缝切换。
- **🔄 HEIC/HEIF 智能兼容**：前端自动识别苹果高效率图像格式，并静默转换为 JPEG 上传，完美解决跨设备黑屏痛点。
- **👥 灵活的权限控制**：支持自定义“最大阅读时长”、“允许查看总人数”（或不限人数的群发模式），并可一键开启“仅限微信内打开”防抓包。
- **🗑️ 数据自清理**：即便图片无人查看，KV 记录与 R2 底层数据也会在 7 天后被平台生命周期规则强制抹除。

## 🚀 部署指南

本项目采用纯单文件架构，无需安装 Node.js、Wrangler 或任何依赖，完全可以在浏览器内点按完成部署。

### 第一步：准备 Cloudflare 存储资源
1. 登录 Cloudflare Dashboard。
2. **创建 R2 存储桶**：
   - 导航至 **R2** -> **Create bucket**。
   - 命名为 `flash-photo-bucket`（可自定义），**禁止开启**公开访问 (Public Access)。
3. **创建 KV 命名空间**：
   - 导航至 **Workers & Pages** -> **KV** -> **Create a namespace**。
   - 命名为 `FLASH_KV`（可自定义）。

### 第二步：创建并配置 Worker
1. 导航至 **Workers & Pages** -> **Create Application** -> **Create Worker**，命名后点击部署。
2. 进入刚刚创建的 Worker 详情页，选择 **Settings (设置)** -> **Bindings (绑定)**。
3. **绑定 R2 存储桶**：
   - 点击 Add Binding -> R2 bucket。
   - **Variable name 必须填入**：`BUCKET`
   - 选择第一步创建的 R2 存储桶。
4. **绑定 KV 命名空间**：
   - 点击 Add Binding -> KV namespace。
   - **Variable name 必须填入**：`VIEW_RECORDS`
   - 选择第一步创建的 KV 命名空间。
5. 点击右下角 Deploy 保存设置。

### 第三步：粘贴代码
1. 在 Worker 详情页点击右上角的 **Edit Code (编辑代码)**。
2. 清空编辑器中的所有默认代码。
3. 将本项目中的 `worker.js` 文件内容完整复制并粘贴进去。
4. 点击右上角的 **Deploy**。

🎉 **部署完成！** 现在你可以点击 Worker 提供的 `*.workers.dev` 或绑定你的自定义域名开始安全分享你的闪照了。

## 🧹 设置 R2 自动清理规则（强烈建议）

为了防止被遗忘的“死数据”长期占用存储空间，建议配置 R2 生命周期规则：
1. 进入你创建的 R2 存储桶，点击 **Settings (设置)**。
2. 找到 **Object lifecycle rules**，点击 **Add rule**。
3. **Rule Name**: `Auto-Delete-7-Days`
4. **Action**: 选择 `Delete uploaded objects`
5. **Age**: 填入 `7` Days。
6. 保存规则。

## 🔒 隐私与安全说明

本程序的设计原则是 **“不信任任何人（包括系统拥有者）”**：
1. 用户上传的数据流直接进入 Worker 内存。
2. Worker 根据随机生成的 UUID 和特定盐值（Salt）派生出一次性 AES 密钥。
3. 数据被 AES-GCM 算法加密后存入物理硬盘（R2）。
4. 用户的查阅 URL 是解开数据的唯一凭证。一旦 KV 计数器耗尽，凭证作废，密文数据和 KV 记录会被立即执行 `delete` 指令删除。
